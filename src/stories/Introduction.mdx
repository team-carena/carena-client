# CareNA Storybook

CareNA Storybook은  
**공용 UI 컴포넌트를 문서화하고, 독립적으로 개발 및 검증하기 위한 공간**입니다.

페이지 구현과 분리된 환경에서  
컴포넌트의 구조와 상태를 명확히 정의하고,
팀 내 UI 컨벤션과 디자인 시스템을 공유하는 것을 목표로 합니다.

---

## Storybook이란?

Storybook은 UI 컴포넌트를 독립적으로 개발하고 검증할 수 있는 도구입니다.

CareNA 팀에서는 다음과 같은 이유로 Storybook을 사용합니다.

- 🧩 **컴포넌트 단위 개발**: 페이지와 관계없이 UI를 독립적으로 개발
- 📖 **문서화**: 컴포넌트의 Props와 사용 예시를 한 곳에서 관리
- 🧪 **테스트**: UI 변경 시 시각적인 변경 여부와 상태 유지 여부를 확인

---

## 프로젝트 아키텍처

CareNA는 FSD(Feature-Sliced Design) 아키텍처를 따릅니다.

```
src/
├── pages/[feature]/     # 기능별 코드
│   ├── components/     # 기능 컴포넌트
│   └── ...
├── shared/      # 공용 컴포넌트, 유틸리티, 디자인 토큰
└── stories/     # Storybook 스토리
```

---

## Story 기본 원칙

### 📄 스토리 작성 위치

모든 컴포넌트의 스토리는 `src/stories/` 디렉토리 내에서 관리합니다.
스토리 구조를 통일성 있게 유지하고, 컴포넌트 문서화를 한 곳에서 집중적으로 관리하기 위함입니다.

### 📌 스토리 작성 규칙

파일명: `컴포넌트명.stories.ts`

- (ex) `Button` 컴포넌트 → `Button.stories.ts`

- 모든 스토리는 **args 기반**으로 작성합니다.
- UI 상태는 props(`variant`, `size`, `disabled` 등)로 명확히 표현합니다.
- 하나의 스토리는 **대표적인 한 가지 상태**를 나타냅니다.

## Storybook 사용법

### 컴포넌트 탐색
- 좌측 사이드바에서 카테고리별로 컴포넌트를 탐색할 수 있습니다.
- 각 컴포넌트는 여러 상태를 나타내는 스토리를 포함합니다.

### Props 조작
- **Controls** 패널을 통해 props 값을 변경하며 UI 변화를 확인합니다.
- 실제 서비스에서 사용하지 않는 props는 스토리에 포함하지 않습니다.

### 문서 확인
- Docs 탭에서 컴포넌트 설명과 사용 예시를 확인합니다.
- Storybook은 컴포넌트 사용법을 공유하는 참고 문서로 활용합니다.
